Supply Chain Issues Review
# README
Overview
This repository documents an AI-assisted workflow for reviewing and communicating supply chain issues. The project focused on turning email inputs, meeting context, and analytical data into structured outputs for leadership review, stakeholder alignment, and follow-up execution.
Workflow
•	Email analysis: Summarized the source email to identify compliance-related supply chain issues and recommended actions.
•	Report generation: Converted the summary into an executive-ready report for leadership review.
•	Meeting preparation: Drafted a meeting request and agenda to review the reported issues with stakeholders.
•	Action planning: Used meeting context and transcript inputs to produce a follow-up action plan.
•	Data analysis: Reviewed component and supplier data to identify at-risk items and supplier performance concerns.
•	Presentation preparation: Created presentation content for executive communication based on the findings and reports.
Deliverables
•	Task 1 - Email Summary Prompt Output.docx: Summary of the original supply chain issue email.
•	Task 1 - Report Generation.docx: Executive-ready report derived from the summarized findings.
•	Task 2 - Meeting Request_ Supply Chain Issues Review.msg: Meeting request and agenda for stakeholder review.
•	Task 2 - Supply Chain Issue Action Plan.docx: Post-meeting action plan based on the agenda and transcript inputs.
•	Task 3 - Supply Chain Issue Analysis Report.docx: Analytical report covering at-risk components and supplier performance.
•	Task 4 - Supply Chain Issues Review – Presentation.pptx: Executive presentation summarizing the overall review and findings.
How AI Was Used
AI was used to accelerate documentation, summarize inputs, generate structured business artifacts, and support communication with leadership and stakeholders. It helped convert raw emails, meeting context, transcripts, and analytical data into actionable outputs such as summaries, reports, agendas, action plans, and presentation content.
